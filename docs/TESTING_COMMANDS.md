# 🧪 Тестовые команды для отладки уведомлений

## Доступные команды

### 1️⃣ `/test_notification` - Тест уведомления учителю

**Описание:** Отправляет тестовое уведомление о завершении занятия (как если бы до конца занятия осталось 5 минут)

**Доступ:** Только для администраторов (ID из `ADMINS_IDS`)

**Что тестирует:**
- ✅ Уведомление учителю о завершении занятия
- ✅ Форматирование даты с днем недели
- ✅ Отображение информации об ученике
- ✅ Статус оплаты
- ✅ Продолжительность занятия
- ✅ Интерактивные кнопки (оплата, ДЗ, завершение)

**Как использовать:**
```
/test_notification
```

**Результат:**
```
🧪 ТЕСТОВОЕ УВЕДОМЛЕНИЕ

⏰ Занятие завершается через 5 минут!

👤 Ученик: Иван Иванов
📅 04.11.2024 (Пн) в 15:00
⏱️ Продолжительность: 1ч
❌ Оплата: не оплачено

📝 Не забудьте отметить статус оплаты и задать домашнее задание!

[💰 Оплачено] [❌ Не оплачено]
[📝 Задать ДЗ] [✅ Занятие завершено]

---

✅ Тестовое уведомление отправлено!

📊 Данные занятия:
• ID: 123
• Дата: 2024-11-04
• Время: 15:00
• Ученик: Иван Иванов (ID: 987654321)

ℹ️ Кнопки работают в реальном режиме - изменения будут сохранены в БД
```

**⚠️ Важно:**
- Использует **ближайшее запланированное занятие** из БД
- Кнопки работают **в реальном режиме** - изменения сохраняются
- Если нет запланированных занятий, команда вернет ошибку

---

### 2️⃣ `/test_student_notification` - Тест уведомления ученику

**Описание:** Отправляет тестовое напоминание ученику о занятии (как если бы до занятия осталось 3 часа)

**Доступ:** Только для администраторов (ID из `ADMINS_IDS`)

**Что тестирует:**
- ✅ Напоминание ученику о занятии
- ✅ Информация о задолженности
- ✅ Домашнее задание (если есть)
- ✅ Формат занятия (онлайн/оффлайн)
- ✅ Форматирование даты с днем недели

**Как использовать:**
```
/test_student_notification
```

**Результат:**
```
🧪 ТЕСТОВОЕ НАПОМИНАНИЕ

📚 Напоминание о занятии

📅 04.11.2024 (Пн) в 15:00
⏱️ Продолжительность: 1ч
📍 Формат: онлайн

💰 Задолженность: 2000₽ за 2 занятий

📖 Домашнее задание: Выполнить упражнения 1-5

---

✅ Тестовое уведомление (для ученика) отправлено!

📊 Данные:
• Ученик: Иван Иванов (ID: 987654321)
• Задолженность: 2000₽
• Неоплаченных занятий: 2

ℹ️ В реальном режиме это сообщение отправляется ученику за 3 часа до занятия
```

**⚠️ Важно:**
- Отправляется **вам** (администратору), а не ученику
- Показывает, как выглядит уведомление для ученика
- Использует **ближайшее запланированное занятие**

---

## 📋 Требования для тестирования

1. **Наличие запланированных занятий:**
   - Должно быть хотя бы одно занятие со статусом `scheduled`
   - Дата занятия должна быть >= сегодня

2. **Права доступа:**
   - Ваш Telegram ID должен быть в списке `ADMINS_IDS` в `config.py`

3. **Бот должен быть запущен:**
   ```bash
   python run.py
   ```

## 🔧 Как создать тестовое занятие

Если у вас нет занятий для тестирования:

### Вариант 1: Через бота (рекомендуется)
1. Создайте расписание для ученика
2. Дождитесь автоматического создания занятий (или перезапустите бота)

### Вариант 2: Напрямую в БД
```sql
INSERT INTO lessons (
    student_id, 
    lesson_date, 
    lesson_time, 
    lesson_format, 
    price, 
    duration, 
    status, 
    payment_status
) VALUES (
    123456789,           -- ID ученика
    date('now', '+1 day'), -- Завтра
    '15:00',             -- 15:00
    'online',            -- Онлайн
    1000,                -- 1000₽
    60,                  -- 60 минут
    'scheduled',         -- Запланировано
    'unpaid'             -- Не оплачено
);
```

## 🐛 Устранение неполадок

### "Нет запланированных занятий для тестирования"

**Причина:** В БД нет занятий со статусом `scheduled` и датой >= сегодня

**Решение:**
1. Создайте расписание через бота
2. Или добавьте занятие вручную в БД
3. Проверьте, что дата занятия >= сегодня

### "⛔ Эта команда доступна только администраторам"

**Причина:** Ваш Telegram ID не в списке `ADMINS_IDS`

**Решение:**
1. Откройте `config.py`
2. Добавьте свой ID в `ADMINS_IDS`:
   ```python
   ADMINS_IDS = [123456789, ВАШ_ID]
   ```
3. Перезапустите бота

### Кнопки не работают

**Причина:** ID занятия некорректен или занятие было удалено

**Решение:**
1. Проверьте, что занятие существует в БД
2. Перезапустите команду `/test_notification`

## 💡 Советы по тестированию

1. **Используйте разные сценарии:**
   - Оплаченное занятие
   - Неоплаченное занятие
   - Занятие с домашним заданием
   - Занятие с задолженностью у ученика

2. **Проверьте работу кнопок:**
   - ✅ Отметить оплату
   - ❌ Отметить неоплату
   - 📝 Задать домашнее задание
   - ✅ Завершить занятие

3. **Проверьте обновление данных:**
   - После нажатия кнопок проверьте, что данные обновились в БД
   - Посмотрите историю занятий через интерфейс бота
