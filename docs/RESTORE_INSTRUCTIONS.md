# Инструкция по восстановлению файлов из ветки main

Если что-то пошло не так после обновления, вы можете вернуть все файлы из ветки `main`.

## Вариант 1: Откатить только измененные файлы (рекомендуется)

Новые файлы (task_executor.py, task_manager.py) останутся, но измененные файлы вернутся к версии из main:

```bash
git checkout main -- bot_template/
```

Это откатит:
- `bot_template/app/handlers.py`
- `bot_template/database/db_manager.py`
- `bot_template/run.py`

Но сохранит новые файлы:
- `bot_template/database/task_executor.py`
- `bot_template/database/task_manager.py`

## Вариант 2: Полностью вернуться к состоянию main

Если нужно полностью удалить все изменения, включая новые файлы:

```bash
# Откатываем измененные файлы
git checkout main -- bot_template/

# Удаляем новые файлы вручную
rm bot_template/database/task_executor.py
rm bot_template/database/task_manager.py
```

## Вариант 3: Переключиться на ветку main

⚠️ **ВНИМАНИЕ**: Это удалит все незакоммиченные изменения!

```bash
git checkout main
```

## Проверка изменений

Перед откатом можно посмотреть, что изменилось:

```bash
# Посмотреть измененные файлы
git status

# Посмотреть конкретные изменения
git diff bot_template/
```

## После отката

После восстановления файлов из main, боты в папке `tutors/` продолжат работать со старой версией кода из `bot_template/`.

---

💡 **Совет**: Перед обновлением на сервере сделайте коммит текущих изменений или создайте бэкап, чтобы можно было вернуться к новой версии при необходимости.
